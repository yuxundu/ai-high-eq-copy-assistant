# 出图 / 出文触发逻辑与提示语表

## 概述
本规范说明在不同使用场景下，系统应输出文字还是生成图片，并列出相应的触发条件与用户提示语。目的是保证即时沟通效率，同时促进内容分享。

## 出文优先场景
以下场景默认优先提供文本输出，用户可直接复制：
- **微信聊天救场**：用户需要立刻回复对方，避免打扰节奏，不自动生成图片。
- **职场沟通**：如邮件、钉钉消息等正式沟通场合，文字更专业。

提示语示例：
- 生成后显示：“已生成3条高情商回复，选一条复制发送吧。”
- 复制成功后显示：“已复制，可直接粘贴发送。”

## 可选生成图场景
用户可在查看文本后选择生成分享卡片图的场景：
- **恋人或朋友沟通**：用户可能想在朋友圈分享心情，可提供“生成卡片图”按钮。
- **社交媒体配文**：如发朋友圈、小红书等，生成图片可增强视觉效果。
- **节日祝福**：节日模板自动提供卡片图选项。

触发逻辑：
- 用户点击“生成卡片图”按钮手动触发。
- 系统在复制后弹出轻提示：“要不要生成一张卡片图？”但不强制。

## 自动生成图场景
系统在特定条件下可自动建议生成图片：
- **节日当天**：如春节、520、情人节等，生成节日主题卡片更符合场景。
- **生成高情绪浓度文案**：系统评估文案情感浓度较高时，提示用户可分享。

提示语示例：
- “这句话很赞，要不要生成一张卡片图？”
- “节日特效背景上线，生成节日卡片 🎉”

## 不生成图的场景
以下场景不自动生成图，并且不建议用户出图：
- **职场正式沟通**：用户需保持专业。
- **私密对话**：涉及隐私，不鼓励用户截图分享。

## UX 文案规范
在触发出图或出文时，使用友好的提示文案：
- 文字生成完成：“已根据你的描述生成多条文案，请选择复制。”
- 手动触发出图：“生成卡片图 🌈”
- 卡片生成成功：“卡片已生成，你的文字更温柔了 🌷”
- 分享引导：“朋友看到这张图，会想问你在哪生成的 😉”
